### Некоторый перечень проблем и их решения ### 

1. При команде "git remote add origin ..." возникает ошибка доступа
	Решение: 
	а) Создать ssh ключ: ssh-keygen ed25519
	б) В настройках пользователя GitHub в разделе "Ключи SSH и GPG" добавить ранее сгенерированный ключ ~/.ssh/id_ed25519.pub в строку "Ключ".
	в) Сохранить изменения
	г) Запустить команду git remote add origin ... заново
Заметка: при работе через ssh ключи необходимо копировать ssh ключ репозитория, а не HTTPS URL

2. При копировании удаленного репозитория выводится сообщение о том, что репозиторий уже существует
	Решение:
	а) Проверить, нет ли в текущем каталоге файла .git
	б) Если файл есть, удалить его
	в) Запустить клонирование репозитория снова
